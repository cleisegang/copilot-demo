Copilot Surfaces

Where GitHub Copilot shows up in your workflow. Click a tab to explore each surface.

🖥️ What it is
  • A terminal-native AI agent — runs directly in your shell
  • Conversational chat interface in the terminal (ask it anything)
  • Autonomous file creation, editing, and command execution
  • Operates on files in the current working directory by default
  • Asks permission before modifying files or running risky commands
  • Available with all Copilot plans (Free, Pro, Business, Enterprise)
⚡ Key use cases
  • Bug fixes and targeted refactoring
  • Implementing incremental features from a description
  • Writing and updating documentation
  • Improving test coverage
  • Prototyping new projects (greenfield)
  • Setting up local dev environments
  • Finding the right CLI command or explaining unknown commands
  • Interacting with GitHub.com (list PRs, create issues)
💬 Slash commands
  • /mcp add — add a new MCP server interactively
  • /mcp show — list configured MCP servers
  • /mcp disable <name> — disable a server
  • /login — authenticate with GitHub
  • /feedback — send feedback to the team
  • /help — show all available commands
📦 Install methods
  • npm (all platforms): npm install -g @github/copilot
  • Homebrew (macOS/Linux): brew install copilot-cli
  • WinGet (Windows): winget install GitHub.Copilot
  • Install script: curl -fsSL https://gh.io/copilot-install | bash
  • Direct download: github.com/github/copilot-cli/releases

📝 Copilot Edits — Edit Mode
  • Propose changes across multiple files from one prompt
  • You choose which files are in scope
  • Review and accept/reject each change individually
  • Best for: quick, targeted, controlled edits
  • Available in VS Code and JetBrains
🤖 Copilot Edits — Agent Mode
  • Autonomous multi-step task execution in your IDE
  • Copilot determines which files to change
  • Can run terminal commands, iterate on errors
  • Integrates with MCP servers for external tools
  • Best for: complex tasks, full autonomy, MCP integration
ℹ️ Agent mode vs. Coding Agent: Agent mode runs locally in your IDE. Copilot Coding Agent runs in GitHub Actions and creates pull requests for background async work. Both are powerful — the difference is local vs. cloud.

GitHub Actions & Copilot

GitHub Actions is the automation backbone behind Copilot's most powerful features — and Copilot can help you author, fix, and optimize your workflows.

⚙️ Copilot Coding Agent runs on Actions
  • Every Coding Agent task spins up an ephemeral GitHub Actions runner
  • Agent gets its own isolated environment with full repo access
  • Runs your test suites, linters, and build scripts to validate changes
  • Full transparency — every step is visible in the Actions logs
  • Consumes your org's existing Actions minutes quota
  • Environment reuses your existing CI workflows automatically
✍️ Copilot writes your workflows
  • Describe the workflow you need in plain English in Copilot Chat or CLI
  • Copilot generates syntactically correct YAML, including triggers, jobs, and steps
  • Ask it to add, modify, or debug specific jobs in existing workflows
  • @workspace /fix on a failing workflow to get targeted suggestions
  • Understands GitHub context: github.* vars, secrets, matrix builds
  • Works in VS Code with inline suggestions as you type .yml
🔌 GitHub MCP Server + Actions tools
  • GitHub MCP Server exposes Actions APIs as Copilot tools
  • List workflow runs, re-run failed jobs, check status — all from Copilot Chat
  • Trigger workflow_dispatch events without leaving your terminal or IDE
  • View workflow run logs to debug CI failures with Copilot's help
  • Available in both Copilot CLI (built-in) and VS Code (remote MCP server)
🔒 Coding Agent environment & secrets
  • Set up a Copilot environment in repo Settings → Copilot → Coding agent
  • Add secrets and variables prefixed with COPILOT_MCP_ for use by MCP servers
  • Coding Agent inherits the GITHUB_TOKEN automatically (scoped to the repo)
  • Configure MCP servers for the agent via JSON in repo settings
  • Organization/enterprise admins can set MCP config for custom agents via YAML frontmatter
💡 How it all connects: You assign a GitHub Issue to Copilot → Coding Agent starts a GitHub Actions job → agent explores code, makes changes, runs your CI → opens a PR. Your existing GitHub Actions workflows are the test harness for every agent task.

Example workflows & prompts

Paste any of these prompts into Copilot Chat (VS Code or GitHub.com) or Copilot CLI to generate a workflow:

# In Copilot CLI or VS Code Chat:
"Create a GitHub Actions workflow that runs Jest tests on every PR targeting main"

"Add a workflow to build and push a Docker image to GHCR on every push to main"

"Write a workflow that runs ESLint and TypeScript type checking on PRs"

"Create a release workflow that tags a version and publishes to npm when I push a git tag"

"Add a nightly workflow to run database migrations against a staging environment"

"Fix the failing step in my CI workflow — the npm install is timing out"

"Explain what the matrix strategy in my workflow.yml does and suggest improvements"

A solid Node.js CI workflow to start from. Ask Copilot to adapt it to your stack.

.github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20, 22]

    steps:
      - uses: actions/checkout@v4

      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build --if-present

Use the GitHub MCP Server in Copilot CLI or VS Code to investigate and fix failing runs without leaving your editor.

Debug failing Actions via Copilot CLI (GitHub MCP built-in)

# Ask Copilot to look at your recent runs:
"List the most recent failed workflow runs on cleisegang/my-repo"

"Show me the logs from the last failed CI run on the main branch"

"What caused the failure in workflow run #12345?"

"Fix the issue in my workflow that's causing the Docker build step to fail"

# and suggest or directly apply fixes to your workflow YAML.

The Coding Agent uses your repo's existing CI. Optionally configure a Copilot environment with secrets and MCP servers for the agent's Actions runs.

Coding Agent MCP config (repo Settings → Copilot → Coding agent)
{
  "mcpServers": {
    "sentry": {
      "type": "local",
      "command": "npx",
      "args": ["@sentry/mcp-server@latest"],
      "tools": ["get_issue_details", "get_issue_summary"],
      "env": {
        "SENTRY_ACCESS_TOKEN": "COPILOT_MCP_SENTRY_ACCESS_TOKEN"
      }
    },
    "playwright": {
      "type": "local",
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "tools": ["*"]
    }
  }
}

// Secrets/variables for the agent must be prefixed with COPILOT_MCP_
// Add them in: repo Settings → Environments → copilot → Secrets/Variables
🔑 The Coding Agent's GITHUB_TOKEN is automatically scoped to the repo it is working in. MCP server secrets must be added to the copilot environment in your repository settings with the COPILOT_MCP_ prefix.

Model Context Protocol (MCP)

MCP is the open standard that connects Copilot to external tools, data sources, and services. Think of it as a plug-in system for your AI assistant.

🔌
What is MCP?
An open standard (modelcontextprotocol.io) that defines how AI models share context with external tools. Connect Copilot to databases, APIs, browsers, and more — without custom integrations.
🐙
GitHub MCP Server
Built-in to Copilot CLI (no setup needed) and available remotely for VS Code. Gives Copilot access to GitHub APIs: repos, issues, PRs, code scanning, Actions, and more.
📋
GitHub MCP Registry Preview
A curated directory of MCP servers at github.com/mcp. Browse and install servers from the community and GitHub partners. Discover tools like Playwright, Sentry, Linear, and more.
🔒
Security
Push protection prevents secrets from appearing in AI-generated responses. Org/Enterprise admins can enable or disable MCP via policy. Toolsets let you limit which tools are available.

MCP Configuration — Copilot CLI

Add MCP server interactively in Copilot CLI
# In Copilot CLI, use the /mcp add command to add a server interactively
/mcp add

# Or edit the config file directly
# Location: ~/.copilot/mcp-config.json
{
  "mcpServers": {
    "playwright": {
      "type": "local",
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": {},
      "tools": ["*"]
    },
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp",
      "headers": { "CONTEXT7_API_KEY": "YOUR-API-KEY" },
      "tools": ["*"]
    }
  }
}

# Manage servers
/mcp show                    # list all servers
/mcp show playwright         # details + tools for one server
/mcp disable context7        # disable without deleting
/mcp edit playwright         # edit config
/mcp delete playwright       # remove server

Quick Start Guide

Get from zero to AI-powered development in minutes. Follow these steps to set up Copilot CLI and unlock the full ecosystem.

1
Get a GitHub Copilot plan
Start for free or upgrade to Pro for unlimited IDE features. Students, teachers, and open source maintainers may qualify for free Pro access.
# Visit to get started for free or choose a plan:
https://github.com/features/copilot/plans

# For students / educators / OSS maintainers:
https://education.github.com/benefits
2
Install Copilot CLI
Choose the install method for your platform. npm works everywhere; Homebrew is recommended on macOS/Linux.
# macOS / Linux — Homebrew (recommended)
brew install copilot-cli

# macOS / Linux — install script
curl -fsSL https://gh.io/copilot-install | bash

# All platforms — npm (requires Node.js 22+)
npm install -g @github/copilot

# Windows — WinGet
winget install GitHub.Copilot
3
Launch & authenticate
Run copilot from any terminal. On first launch you'll be prompted to authenticate via GitHub's device flow. Takes about 30 seconds.
# Launch Copilot CLI
copilot

# If not already authenticated, use /login in the interactive session
/login

# Or set an environment variable for non-interactive auth:
export COPILOT_GITHUB_TOKEN="your_fine_grained_pat"
# (PAT needs "Copilot Requests" permission)
4
Start your first session
Navigate to your project and describe what you want to do in plain English. Copilot will propose changes, ask for confirmation before modifying files, and iterate based on your feedback.
# Navigate to your project
cd my-project

# Launch Copilot CLI
copilot

# Example prompts to try:
"Explain what this codebase does"
"Add input validation to the user registration form"
"Write unit tests for the auth module"
"Fix the TypeScript error in src/api/users.ts"
"Set up a GitHub Actions CI workflow"
"Refactor the database module to use async/await"
5
Add your first MCP server
Extend Copilot's capabilities with MCP servers. The GitHub MCP Server is built in. Add additional servers like Playwright for browser automation, or any tool in the MCP Registry.
# The GitHub MCP Server is already built in — no setup needed!

# Add Playwright MCP server for browser automation:
/mcp add
# → Server Name: playwright
# → Server Type: STDIO
# → Command: npx @playwright/mcp@latest

# Or add it directly to ~/.copilot/mcp-config.json:
{
  "mcpServers": {
    "playwright": {
      "type": "local",
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "tools": ["*"]
    }
  }
}

# Browse available MCP servers:
# https://github.com/mcp
6
Set up VS Code
Install the GitHub Copilot extension in VS Code, sign in with the same GitHub account, and you'll have inline suggestions, Copilot Chat, Edits, and agent mode available immediately.
# Install GitHub Copilot extension from VS Code marketplace:
# https://marketplace.visualstudio.com/items?itemName=GitHub.copilot

# Or install via CLI:
code --install-extension GitHub.copilot
code --install-extension GitHub.copilot-chat

# Open VS Code and sign in via the Accounts menu (bottom-left)
# Use the same GitHub account with your Copilot plan

# Enable agent mode in Copilot Chat:
# Click the Copilot icon → Chat → select "Agent" from dropdown

# Add GitHub MCP Server to .vscode/mcp.json:
{
  "servers": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/"
    }
  }
}
🎉 You're all set! Ask Copilot CLI to help with any task in your terminal. Open Copilot Chat in VS Code for IDE-integrated assistance. Assign GitHub Issues to the Coding Agent for background autonomous work. The whole ecosystem works with one GitHub account.
